About

SAGA defines a high-level access mechanisms for distributed infrastructure components like job schedulers, file transfer and resource provisioning services. Given the heterogeneity of distributed infrastructure, SAGA provides a much needed interoperability layer that lowers the complexity and improves the simplicity of using distributed infrastructure whilst enhancing the sustainability of distributed applications, services and tools.

saga-python provides a Python module that is compliant with the OGF GFD.90 SAGA (Simple API for Grid Applications) specification. Behind the API fa√ßade, saga-python implements a flexible adaptor architecture. Adaptors are dynamically loadable modules that interface the API with different middleware systems and services. Most application developers use the adaptors that are already part of saga-python, but you can easily implement your own in case your backend system is not supported yet.

saga-python's main focus is ease of use and simple user-space deployment in heterogeneous distributed computing environments. It supports a wide range of application use-cases from simple, uncoupled tasks to complex workflows. saga-python is being used on many distributed cyberinfrastructures, including XSEDE, OSG and FutureGrid where it marshals tens of thousands of cores and moves terabytes of data in production science applications.

Supported Middleware

Extensibility through adaptors.

Saga-python supports a wide range of distributed computing middleware and systems via a flexible adaptor architecture.

The latest saga-python release install with the following adaptors:

  • SSH and GSISSH
  • Condor and Condor-G
  • PBS and Torque
  • Sun Grid Engine
  • SLURM

Besides mapping the heterogeneous backend interfaces to the SAGA API, the majority of adaptors also provide transparent remote access to otherwise exclusively local services (like most HPC queuing systems) by tunneling interface calls through either SSH or GSISSH.

A list of available adaptors and their description can be found in the manual.

Get Started

Run your first saga-python program in less then 5 minutes.

Installation

The easiest way to install saga-python under your user account is via virtualenv and easy_install or pip. 

$ virtualenv $HOME/saga
$ source $HOME/saga/bin/activate
$ easy_install saga-python

More detailed installation instructions can be found in the manual.

Example Script

This example connects to a remote PBS cluster via SSH and runs a simple /bin/sleep job. More information about the saga-python job API can be found in the respective manual section.

If you don't have access to a PBS cluster, you can try a different adaptor by changing the url scheme in line 9. For example, you can use slurm[+ssh]://... to access a SLURM-based cluster, or condor[+ssh]://... to access a Condor(-G) gateway. A full list of available adaptors and their URL scheme can be found in the manual.

import saga

if __name__ == "__main__":

    try:
        # Create a job service object that represent a remote pbs cluster.
        # The keyword 'pbs' in the url scheme triggers the PBS adaptors
        # and '+ssh' enables PBS remote access via SSH.
        js = saga.job.Service("pbs+ssh://alamo.futuregrid.org")

        # Next, we describe the job we want to run. A complete set of job 
        # description attributes can be found in the API documentation. 
        jd = saga.job.Description()
        jd.queue           = 'batch'
        jd.environment     = {'RUNTIME': '10'}
        jd.wall_time_limit = 1 # minutes
        jd.executable      = '/bin/sleep'
        jd.arguments       = ['$RUNTIME']

        # Create a new job from the job description. The initial state of 
        # the job is 'New'.
        sleepjob = js.create_job(jd)

        # Now we can start our job.
        sleepjob.run()

        print "Job ID      : %s" % (sleepjob.id)
        print "Job State   : %s" % (sleepjob.state)

        # Wait for our job to finish (or fail).
        sleepjob.wait()

        print "Job State   : %s" % (sleepjob.state)
        print "Exitcode    : %s" % (sleepjob.exit_code)

    except saga.SagaException, ex:
        # Catch all saga exceptions
        print "An exception occured: (%s) %s " % (ex.type, (str(ex)))
        # Trace back the exception. That can be helpful for debugging.
        print " \n*** Backtrace:\n %s" % ex.traceback

Paste the above script into a file and run it. Your output should look similar to this: 

$ python ./sleepjob.py
Job ID      : [pbs+ssh://alamo.futuregrid.org]-[209181]
Job State   : Pending
Job State   : Done
Exitcode    : 0
