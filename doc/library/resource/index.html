3. Resource Management¶

The basic usage of the resource module is as follows:

.. literalinclude:: ../../../examples/resource/amazon_ec2.py

See also

More examples on how to use the SAGA resource module can be found in the Resource Module section of the Code Examples chapter.

Like all SAGA modules, the resource module relies on middleware adaptors to provide bindings to a specific resource manager. Adaptors are implicitly selected via the scheme part of the URL, e.g., local:// in the example above selects the local resource adaptor.

Note

A list of available adaptors and supported resource managers can be found in the Developer Documentation part of this documentation.

3.1. Resource manager – saga.resource.Manager¶

todo:Describe how to work with resource managers.
class saga.resource.Manager(*pargs, **pkwargs)¶

Bases: saga.base.Base, saga.async.Async

In the context of SAGA-Python, a ResourceManager is a service which asserts control over a set of resources. That manager can, on request, render control over subsets of those resources (resource slices) to an application.

This Manager class represents the contact point to such ResourceManager instances – the application can thus acquire compute, data or network resources, according to some resource specification, for a bound or unbound amount of time.
